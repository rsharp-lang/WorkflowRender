create_memory_context {WorkflowRender} R Documentation

Create an In-Memory Workflow Context for Debugging

Description

Creates a lightweight in-memory workflow context object that is intended for debugging and unit testing scenarios where no on-disk workspace is required. All workspace directory paths of the created context are set to \code{NULL}, and the optional \code{mounts} callback can be used to register application modules into the workflow registry immediately after the context is created.

Usage

create_memory_context(
    mounts = NULL,
    ssid = NULL);

Arguments

mounts

an optional callable function that is invoked (with no arguments) after the in-memory context has been installed. This function is expected to mount the application modules into the workflow registry by calling \code{\link{hook}} for each module that should participate in the workflow.

ssid

the session identifier used for the multi-user environment. The default value of \code{NULL} refers to the global user environment.

Details

The created context has the same structure as the one produced by \code{\link{init_context}}, but with all directory paths set to \code{NULL}. This makes the function particularly useful for dependency-graph analysis (see \code{\link{dependency_graph}}) and for unit tests that do not need to touch the filesystem.

Authors

xieguigang

Value

Invisibly returns \code{NULL}. The side effect of this function is the installation of an in-memory workflow context object in the global environment.

Examples

#\dontrun{

create_memory_context(function() {
hook(my_analysis_module);
});

#}
