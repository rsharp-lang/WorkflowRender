app_check.delegate {WorkflowRender} R Documentation

Check the Application Module Object Signature

Description

Validates that the supplied application object has the correct structure to be registered into the workflow context. The function checks that the object is a list with the expected slots and that the \code{call} slot is a callable function.

Usage

app_check.delegate(analysis);

Arguments

app

the application object to be validated.

Details

This function is invoked by \code{\link{hook}} to ensure that the supplied application object was constructed via the \code{\link{app}} function (or via \code{\link{build_app}}) and therefore has the expected structure. When the check fails, the workflow is aborted via \code{\link{throw_err}} with an appropriate error message.

Authors

xieguigang

Value

A logical value that indicates whether the supplied application object has the correct signature. The value \code{TRUE} is returned when the object passes all of the following checks: \itemize{ \item it is a list object; \item it contains the slots \code{name}, \code{call}, \code{desc}, \code{dependency} and \code{disable}; \item the \code{call} slot is a callable function. } The value \code{FALSE} is returned otherwise.
